Core Types

Documentation for Core Types

Core Types

Position
CORE

Represents a position in the source code (1-indexed line and column). Used to pinpoint exact locations for error reporting and CST data.

Location
CORE

Represents a range in the source code from a beginning position to an ending position. Every AST node has a location that spans the source text it was parsed from.

Core Types

AstName
CORE

Represents a simple name/identifier in the AST. Used for variable names, function names, type names, etc.

Example
-- "myVar" in:
local myVar = 1
-- "func" in:
function func() end
AstLocal
CORE

Represents a local variable declaration in the AST. Tracks the variable's name, location, shadowing information, scope depth, and optional type annotation.

Example
local x = 1 -- "x" is the AstLocal
local y: number -- "y" is the AstLocal (with annotation)
local function f(arg) end -- "f" and "arg" are AstLocals
Fields
name string
The name of the local variable
location Location
Source location of the variable declaration
shadow AstLocal?
Reference to a shadowed local variable with the same name (if any)
functionDepth number
The nesting depth of the function containing this local
loopDepth number
The nesting depth of loops containing this local
annotation AstType?
Optional type annotation for the local variable

Core Types

AstNode CORE Union Type

Union type representing all possible AST nodes that can be stored in the CST map. This includes statements, expressions, types, locals, and various helper nodes.
