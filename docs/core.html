Core Types

Fundamental types used throughout the parser for positions, locations, and identifiers.

Position
HELPER

Represents a position in source code with 1-indexed line and column numbers. Used for error reporting, CST node positions, and token locations.

Fields
linenumber
Line number (1-indexed)
columnnumber
Column number (1-indexed)
Location
HELPER

Represents a range in source code spanning from a beginning position to an ending position. Every AST node has a location field indicating the source text it was parsed from.

Fields
beginPosition
Starting position of the range (inclusive)
end_Position
Ending position of the range (exclusive). Named end_ to avoid Luau keyword conflict.
AstName
ASTHELPER

A simple identifier name used throughout the AST for variable names, function names, type names, field names, etc.

Fields
valuestring
The identifier string
AstLocal
ASTHELPER

Represents a local variable declaration. Tracks the variable's name, source location, shadowing relationships, scope depths, and optional type annotation. Referenced by AstExprLocal when the variable is used.

Fields
namestring
The variable name
locationLocation
Source location of the variable declaration
shadowAstLocal?optional
Reference to another local variable with the same name that this one shadows. nil if no shadowing.
functionDepthnumber
Nesting depth of the function containing this local (0 for top-level)
loopDepthnumber
Nesting depth of loops containing this local
annotationAstType?optional
Type annotation for the local variable (e.g., local x: number)
