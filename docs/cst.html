CST Nodes

CstNode

Union CST

Union type representing all possible Concrete Syntax Tree nodes. CST nodes store positional data for syntactic elements (commas, keywords, etc.) that are not semantically significant but needed for source-accurate transformations.

CST Nodes

CstStatBlock

StatementCST

CST data for a block statement.

Example
do
    print(1); -- Capture semicolon position
    print(2)
end
Fields
bodyCommaPositions {Position}
Positions of semicolons between statements (if any)

CstStatRepeat

StatementCST

CST data for a repeat statement.

Example
repeat
    ...
until cond -- Capture position of 'until'
Fields
untilPosition Position
Position of the `until` keyword

CstStatDo

StatementCST

CST data for a do statement.

Example
do
    ...
end -- Capture position of 'end'
Fields
endPosition Position
Position of the `end` keyword

CstStatFor

StatementCST

CST data for a numeric for loop.

Example
for i: number = 1, 10, 2 do end
-- Captures positions of ':', '=', first ',', second ','
Fields
annotationColonPosition Position?
Position of `:` before loop variable type annotation
equalsPosition Position
Position of the `=` sign
endCommaPosition Position
Position of comma after `to` expression
stepCommaPosition Position?
Position of comma before `step` expression (if present)

CstStatForIn

StatementCST

CST data for a generic for loop.

Example
for i: number, v in pairs(t) do end
-- Captures positions of ':', ',' and ','
Fields
varsAnnotationColonPositions {Position?}
Positions of `:` before variable type annotations
varsCommaPositions {Position}
Positions of commas between loop variables
valuesCommaPositions {Position}
Positions of commas between iterator expressions

CstStatFunction

StatementCST

CST data for a function declaration statement.

Example
function f() end -- Capture position of 'function'
Fields
functionKeywordPosition Position
Position of the `function` keyword

CstStatLocalFunction

StatementCST

CST data for a local function declaration.

Example
local function f() end
-- Captures positions of 'local' and 'function'
Fields
localKeywordPosition Position
Position of the `local` keyword
functionKeywordPosition Position
Position of the `function` keyword

CstStatLocal

StatementCST

CST data for a local variable declaration.

Example
local x: number, y = 1, 2
-- Captures positions of ':', ',' and ','
Fields
varsAnnotationColonPositions {Position?}
Positions of `:` before variable type annotations
varsCommaPositions {Position}
Positions of commas between variable names
valuesCommaPositions {Position}
Positions of commas between initializer expressions

StatementCST

CST data for an assignment statement.

Example
x, y = 1, 2
-- Captures positions of ',', '=' and ','
Fields
varsCommaPositions {Position}
Positions of commas between l-values
equalsPosition Position
Position of the `=` sign
valuesCommaPositions {Position}
Positions of commas between r-values

CstStatCompoundAssign

StatementCST

CST data for a compound assignment statement.

Example
x += 1 -- Capture position of '+='
Fields
opPosition Position
Position of the compound operator (+=, -=, etc.)

CstStatReturn

StatementCST

CST data for a return statement.

Example
return 1, 2 -- Capture position of ','
Fields
commaPositions {Position}
Positions of commas between return values

CstStatTypeAlias

StatementTypeCST

CST data for a type alias declaration.

Example
type Box<T> = { val: T }
-- Captures positions of 'type', '<', '>', '='
Fields
typeKeywordPosition Position
Position of the `type` keyword
genericsOpenPosition Position?
Position of `<` for generics (if present)
genericsCommaPositions {Position}
Positions of commas between generic parameters
genericsClosePosition Position?
Position of `>` for generics (if present)
equalsPosition Position
Position of the `=` sign

CstStatTypeFunction

StatementTypeCST

CST data for a type function declaration.

Example
type function f() end
-- Captures positions of 'type' and 'function'
Fields
typeKeywordPosition Position
Position of the `type` keyword
functionKeywordPosition Position
Position of the `function` keyword

CstExprFunction

ExpressionCST

CST data for a function expression.

Example
function <T>(a: number): string end
-- Captures positions of function, <, >, :, ->
Fields
functionKeywordPosition Position
Position of the `function` keyword
openGenericsPosition Position?
Position of `<` for generics (if present)
genericsCommaPositions {Position}
Positions of commas between generic parameters
closeGenericsPosition Position?
Position of `>` for generics (if present)
argsAnnotationColonPositions {Position?}
Positions of `:` before parameter type annotations
argsCommaPositions {Position}
Positions of commas between parameters
varargAnnotationColonPosition Position?
Position of `:` before vararg type annotation
returnSpecifierPosition Position?
Position of `:` or `->` before return type

CstExprTable

ExpressionCST

CST data for a table constructor expression.

Example
{ a = 1, b = 2 }
Fields
items {CstExprTableItem}
CST data for each table item

CstExprTableItem

ExpressionCST

CST data for a single table item.

Fields
kind "Record" | "General" | "List"
The item kind ("Record", "General", or "List")
equalsPosition Position?
Position of `=` for Record/General items
separator number?
The separator character code (`,` or `;`)
separatorPosition Position
Position of the separator
indexerOpenPosition Position?
Position of `[` for General items
indexerClosePosition Position?
Position of `]` for General items

CstExprIfElse

ExpressionCST

CST data for an if-else expression.

Example
if x then y else z
-- Captures positions of 'then' and 'else'
Fields
thenPosition Position
Position of the `then` keyword
elsePosition Position
Position of the `else`/`elseif` keyword
isElseIf boolean
True if this is an `elseif` branch

CstExprInterpString

ExpressionCST

CST data for an interpolated string expression.

Example
`Value: {x}, Count: {y}`
-- Captures each string segment position
Fields
sourceStrings {string}
The raw source strings (before escape processing)
stringPositions {Position}
Positions of each string segment

CstExprConstantString

ExpressionCST

CST data for a string constant expression. 2=QuotedRaw ([[]]), 3=QuotedInterp (``)

Example
"hello", 'world', [=[text]=]
Fields
sourceString string?
The raw source string (before escape processing)
quoteStyle number
The CST quote style: 0=QuotedSingle (''), 1=QuotedDouble (""),
blockDepth number
The bracket depth for long strings (e.g., 2 for [==[]==]), 0 for quoted strings

CstTypeInstantiation

TypeCST

CST data for type instantiation syntax: `<<T, U>>`.

Example
f<<T, U>>() -- Captures <<, >> and , positions
Fields
leftArrow1Position Position
Position of first `<` character
leftArrow2Position Position
Position of second `<` for `<<`
rightArrow1Position
Position of first `>` character
rightArrow2Position
Position of second `>` for `>>`
commaPositions
Positions of commas between type arguments

CstExprCall

ExpressionCST

CST data for a function call expression.

Example
f(a, b) -- Captures (, ) and ,
Fields
openParens Position?
Position of `(` (nil for string/table calls)
closeParens Position?
Position of `)` (nil for string/table calls)
commaPositions {Position}
Positions of commas between arguments
explicitTypes CstTypeInstantiation?
CST data for explicit type instantiation (if present)

CstExprIndexExpr

ExpressionCST

CST data for a bracketed index expression.

Example
t[k] -- Captures [ and ]
Fields
openBracketPosition Position
Position of `[`
closeBracketPosition Position
Position of `]`

123, 0xFF, 1e10

CST data for unary/binary operator expressions.

Position of the operator

Positions of commas between type parameters

Position of `>` for type parameters

Position of the `...` after the name

PackTypeCST

CST data for an explicit type pack.

Example

Position of `(`
closeParenthesesPosition Position?
Position of `)`

TypeCST

CST data for a function type.

Example
<T>(a: number) -> string -- Captures <, >, (, ), :, -> and ,

Position of `<` for generics

Positions of commas between generic parameters
closeGenericsPosition Position?
Position of `>` for generics

Position of `(` for parameters
argumentNameColonPositions {Position?}
Positions of `:` after parameter names
argumentsCommaPositions {Position}
Positions of commas between parameters
closeArgsPosition Position
Position of `)` for parameters
returnArrowPosition Position
Position of `->` return type separator

CstTypeTable

TypeCST

CST data for a table type.

Example
{ x: number } -- Captures {, }, :
Fields
items {CstTypeTableItem}
CST data for each table item
isArray boolean
True if this is an array shorthand type `{T}`

CstTypeTableItem

TypeCST

CST data for a table type item.

Example
prop: Type, [Idx]: Type
-- Captures :, ,, [, ]
Fields
kind "Property" | "StringProperty" | "Indexer"
"Property", "StringProperty" or "Indexer"
colonPosition Position?
Position of `:` after property name
separator number?
The separator character code
separatorPosition Position?
Position of the separator
indexerOpenPosition Position?
Position of `[` for indexers
indexerClosePosition Position?
Position of `]` for indexers
stringInfo CstExprConstantString?
String constant info for string properties
stringPosition Location?
Location of string key
equalsPosition Position?
Position of `=` for string properties
