CST Nodes

Concrete Syntax Tree data for preserving source-accurate formatting and positions.

CST nodes store positional information for syntactic elements that are not semantically significant (like keywords, commas, and parentheses) but are required for lossless source transforms.

Statement CSTs

CstStatBlock
CSTSTAT

CST data for a block statement.

Fields
bodyCommaPositions{Position}
Positions of semicolons between statements (if any).
CstStatFor
CSTSTAT

CST data for a numeric for loop.

Fields
annotationColonPositionPosition?optional
Position of : before loop variable type annotation.
equalsPositionPosition
Position of the = sign.
endCommaPositionPosition
Position of comma or to keyword alignment.

Expression CSTs

CstExprFunction
CSTEXPR

CST data for a function expression.

Fields
functionKeywordPositionPosition
Position of the function keyword.
openGenericsPositionPosition?optional
argsCommaPositions{Position}
Positions of commas between parameters.
CstExprConstantString
CSTEXPR

CST data for a string constant expression.

Fields
sourceStringstring?optional
The raw source text of the string literal (including quotes/brackets).
quoteStylenumber
The CST quote style: 0=Single(''), 1=Double(""), 2=Raw([[]]), 3=Interp(``).
blockDepthnumber
The bracket depth for long strings (e.g., 2 for [==[]==]).
CstExprConstantNumber
CSTEXPR

CST data for a numeric constant expression.

Fields
valuestring
The raw source text of the number literal (e.g., "0x123", "1.5e-10").
