Helpers & Result

Documentation for Helpers & Result

Helpers

AstAttr
HELPER

A function attribute: `@native`, `@checked`, `@deprecated`, etc. function add(a, b) return a + b end use = "newFunc", reason = "`oldFunc` is empty; `newFunc` is better.", }] function oldFunc() end

Fields
type string
The attribute type ("Native", "Checked", "Deprecated")
args {AstExpr}
Arguments passed to the attribute (for @deprecated)
name string?
The raw attribute name from source
AstGenericType
HELPER

A generic type parameter in a generic declaration: `<T, U = Default>`.

Example
type Box<T> = { value: T } -- "T" is the GenericType
function id<T>(x: T): T end -- "T" is the GenericType
Fields
name string
The name of the generic type parameter
defaultValue AstType?
Optional default type value
AstGenericTypePack
HELPER

A generic type pack parameter in a generic declaration: `<T...>`.

Example
type Map<T...> = () -> T... -- "T..." is the GenericTypePack
Fields
name string
The name of the generic type pack parameter
defaultValue AstTypePack?
Optional default type pack value
AstTypeList
HELPER

A list of types with an optional variadic tail. Used for function parameter and return types.

Example
-- In: (number, string) -> boolean
-- "number, string" form the AstTypeList
Fields
types {AstType}
The list of types
tailType AstTypePack?
Optional variadic tail type pack
AstTypeOrPack
HELPER

A wrapper for either a type or a type pack. Used in generic type arguments where either could be valid.

Example
type Res<T, Packs...> = ...
-- `T` fits in `type`, `Packs...` fits in `typePack`
Fields
type AstType?
The type (if this is a type argument)
typePack AstTypePack?
The type pack (if this is a type pack argument)
AstArgumentName
HELPER

A named function argument (for documentation purposes).

Example
type Cb = (a: string, b: number) -> ()
-- "a" and "b" are AstArgumentName nodes
Fields
name string
The argument name
location Location
Location of the argument name
Binding
HELPER

A binding in a local declaration or function parameter.

Example
local x: number = 1 -- "x: number" is the Binding
function f(val: string) end -- "val: string" is the Binding
Fields
name AstName
The variable name
location Location
Location of the binding
annotation AstType?
Optional type annotation
colonPosition Position?
Position of the `:` before type annotation
BindingList
HELPER

A list of bindings (used in local declarations).

AstDeclaredExternTypeProperty
HELPER

A property declaration in an extern type.

Fields
name string
The property name
nameLocation Location
Location of the property name
ty AstType
The property type
isMethod boolean
True if this is a method (has implicit self)
location Location
Location of the entire property declaration
AstTableProp
HELPER

A property in a table type.

Fields
name AstName
The property name
location Location
Location of the property declaration
type AstType
The property type
access string
Access specifier ("read", "write", or "readwrite")
accessLocation Location?
Location of the access specifier (if present)
AstTableIndexer
HELPER

An indexer in a table type: `[KeyType]: ValueType`.

Fields
indexType AstType
The key/index type
resultType AstType
The value type
location Location
Location of the indexer declaration
access string
Access specifier ("read", "write", or "readwrite")
accessLocation Location?
Location of the access specifier (if present)

Parser Result

ParseError
COREError

Represents a parse error with location and message.

Fields
location Location
Where the error occurred
message string
Human-readable error description
Options
CORE

Parser configuration options.

Fields
captureComments boolean
If true, capture comment locations in the result
storeCstData boolean
If true, store CST data for source-accurate transforms
Lexeme
CORE

Represents a lexer token with type and location.

Fields
type
The token type (see Types enum in parser)
location
The source location of the token
Attrs
CORE

Shorthand for an array of attributes.

Comment
CORE

Represents a captured comment.

Fields
type number
The comment type (single-line or block)
location Location
The source location of the comment
HotComment
CORE

Represents a hot comment (special directive comment like --!strict).

Fields
header boolean
True if this is a header comment (at the start of the file)
content string
The content of the hot comment (after --!)
location Location
The source location of the hot comment
FunctionState
CORE

Internal parser state for tracking function context.

Fields
vararg boolean
True if the current function accepts varargs
loopDepth number
Current nesting depth of loops
Result
CORE

The result of parsing a Luau source file.

Fields
root AstStatBlock
The root AST node (top-level block)
commentLocations {Comment}
List of all captured comments
hotcomments {HotComment}
List of all hot comments (directives)
cstNodeMap {[AstNode]: CstNode}
Map from AST nodes to their CST data
errors {ParseError}
List of all parse errors encountered
