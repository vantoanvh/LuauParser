AST Nodes

Documentation for AST Nodes

Core AST

AstName

AST

Represents a simple name/identifier in the AST. Used for variable names, function names, type names, etc.

Example
-- "myVar" in:
local myVar = 1
-- "func" in:
function func() end
Fields
value string
The source string of the name

AstLocal

AST

Represents a local variable declaration in the AST. Tracks the variable's name, location, shadowing information, scope depth, and optional type annotation.

Example
local x = 1 -- "x" is the AstLocal
local y: number -- "y" is the AstLocal (with annotation)
local function f(arg) end -- "f" and "arg" are AstLocals
Fields
name string
The name of the local variable
location Location
Source location of the variable declaration
shadow AstLocal?
Reference to a shadowed local variable with the same name (if any)
functionDepth number
The nesting depth of the function containing this local
loopDepth number
The nesting depth of loops containing this local
annotation AstType?
Optional type annotation for the local variable

Expressions

AstExpr

Union Expression AST

Union type representing all possible expression nodes in the AST. Expressions are constructs that evaluate to a value.

AstExprGroup AstExprConstantNil AstExprConstantBool AstExprConstantNumber AstExprConstantString AstExprLocal AstExprGlobal AstExprVarargs AstExprCall AstExprIndexName AstExprIndexExpr AstExprFunction AstExprTable AstExprUnary AstExprBinary AstExprTypeAssertion AstExprIfElse AstExprInterpString AstExprInstantiate AstExprError

Statements

AstStat

Union Statement AST

Union type representing all possible statement nodes in the AST. Statements are constructs that perform actions but don't produce values.

AstStatBlock AstStatIf AstStatWhile AstStatRepeat AstStatBreak AstStatContinue AstStatReturn AstStatExpr AstStatLocal AstStatFor AstStatForIn AstStatAssign AstStatCompoundAssign AstStatFunction AstStatLocalFunction AstStatTypeAlias AstStatTypeFunction AstStatDeclareGlobal AstStatDeclareFunction AstStatDeclareExternType AstStatError

Type Packs

AstTypePack

Union Pack Type AST

Union type representing all possible type pack nodes. Type packs represent multiple types, used for function return types and variadic arguments.

AstTypePackExplicit AstTypePackVariadic AstTypePackGeneric

Type Annotations

AstType

Union Type AST

Union type representing all possible type annotation nodes. Type annotations specify the expected types of values in Luau's type system.

AstTypeReference AstTypeTable AstTypeFunction AstTypeTypeof AstTypeUnion AstTypeIntersection AstTypeSingletonBool AstTypeSingletonString AstTypeGroup AstTypeError AstTypeOptional

Core AST

AstNode

Union AST

Union type representing all possible AST nodes that can be stored in the CST map. This includes statements, expressions, types, locals, and various helper nodes.

AstStat AstExpr AstType AstLocal AstArgumentName AstGenericType AstGenericTypePack AstTypePack AstTypeOrPack AstAttr

Expressions

AstExprGroup

ExpressionAST

A parenthesized expression: `(expr)`. Used to override operator precedence or for clarity.

Example
local x = (1 + 2) * 3
local y = (a and b) or c
Fields
expr AstExpr
The inner expression wrapped in parentheses

AstExprConstantNil

ExpressionAST

The `nil` literal constant. Represents the absence of a value in Luau.

Example
local null = nil
Fields
location Location
the `nil` location

AstExprConstantBool

ExpressionAST

A boolean literal constant: `true` or `false`.

Example
local t = true
local f = false
Fields
value boolean
The boolean value (true or false)

AstExprConstantNumber

ExpressionAST

A numeric literal constant (integer or floating-point). The original source text is stored in the CST node (CstExprConstantNumber.value). "HexOverflow" = hex literal exceeds 64 bits, "BinOverflow" = binary literal exceeds 64 bits, "Malformed" = failed to parse (typically produces an error node instead)

Example
local a = 123
local b = 3.14
local c = 0xFF
local d = 1e-10
Fields
value number
The parsed numeric value
parseResult string
Parsing status: "Ok" = success, "Imprecise" = exceeds 2^53 (loses f64 precision),

AstExprConstantString

ExpressionAST

A string literal constant (single-quoted, double-quoted, or long bracket string). The original source string (before escape processing) is stored in CstExprConstantString.sourceString. 1=QuotedSingle (single quotes), 2=QuotedRaw (long brackets [[]]), 3=Unquoted (identifier-style key in table constructor like {foo = 1})

Example
local s1 = "hello"
local s2 = 'world'
local s3 = [[long string]]
Fields
value string
The processed string value (with escape sequences resolved)
quoteStyle number
The quote style: 0=QuotedSimple (double quotes or interp string),

AstExprLocal

ExpressionAST

A reference to a local variable.

Example
local a = 1
print(a) -- ExprLocal referring to 'a'
Fields
local
The AstLocal representing the referenced variable
upvalue boolean
True if this is an upvalue (captured from an outer scope)

AstExprGlobal

ExpressionAST

A reference to a global variable.

Example
print("hello") -- ExprGlobal referring to 'print'
a = 1 -- ExprGlobal referring to 'a' if not defined locally
Fields
name string
The name of the global variable

AstExprVarargs

ExpressionAST

A reference to the varargs `...` in a vararg function. Only valid inside functions declared with `...` parameter.

Example
local function f(...)
    print(...) -- ExprVarargs
end

AstExprCall

ExpressionAST

A function call expression: `func(args)` or `obj:method(args)`.

Example
print("hello")
math.max(1, 2)
object:method()
wrapper { data = 1 } -- Function call without parens (table arg)
Fields
func AstExpr
The expression being called (function or callable object)
args {AstExpr}
The list of argument expressions
self boolean
True if this is a method call using `:` syntax
argLocation Location
The location spanning the argument list (including parentheses)
typeArguments {AstTypeOrPack}?
Optional explicit type arguments for generic functions

AstExprIndexName

ExpressionAST

A named field access expression: `expr.name` or `expr:name`.

Example
t.x
math.pi
obj:method -- Even method access via : creates an index expression (usually part of a call)
Fields
expr AstExpr
The expression being indexed
index string
The name of the field being accessed
indexLocation Location
The location of the field name
opPosition Position
The position of the `.` or `:` operator
op number
The operator character code (46 for `.`, 58 for `:`)

AstExprIndexExpr

ExpressionAST

A bracketed index expression: `expr[index]`.

Example
t[1]
t["key"]
t[variable]
Fields
expr AstExpr
The expression being indexed
index AstExpr
The expression used as the index/key

AstExprFunction

ExpressionAST

A function expression (anonymous function or lambda): `function(...) ... end`.

Example
local f = function(a, b) return a + b end
local g = function(...: number) end
Fields
attributes {AstAttr}
Function attributes like @native, @checked, @deprecated
generics {AstGenericType}
Generic type parameters (e.g., `<T, U>`)
genericPacks {AstGenericTypePack}
Generic type pack parameters (e.g., `<T...>`)
self AstLocal?
The implicit `self` parameter for methods (nil for regular functions)
args {AstLocal}
The list of function parameters
returnAnnotation AstTypePack?
Optional return type annotation
vararg boolean
True if the function accepts varargs `...`
varargLocation Location
Location of the `...` token (if vararg)
varargAnnotation AstTypePack?
Optional type annotation for varargs
body AstStatBlock
The function body (block of statements)
functionDepth number
Nesting depth of this function
debugname string
Debug name for the function (inferred from context)
argLocation Location?
Location of the parameter list

AstExprTable

ExpressionAST

A table constructor expression: `{ items }`.

Example
local t1 = {}
local t2 = {1, 2, 3}
local t3 = {x = 10, y = 20}
local t4 = {["key"] = "value"}
Fields
items {AstExprTableItem}
The list of table items (key-value pairs or list elements)

AstExprTableItem

ExpressionAST

Represents a single item in a table constructor.

Example
{ 1 } -- kind="List", value=1
{ x = const } -- kind="Record", key=x, value=const
{ [expr] = v } -- kind="General", key=expr, value=v
Fields
kind "List" | "Record" | "General"
"List" for array-style `value`, "Record" for `key = value`, "General" for `[key] = value`
key AstExpr?
The key expression (nil for List items)
value AstExpr
The value expression

AstExprUnary

ExpressionAST

A unary operator expression: `-expr`, `not expr`, or `#expr`.

Example
local neg = -x
local inv = not x
local len = #x
Fields
op number
The unary operator (0=not, 1=minus, 2=length)
expr AstExpr
The operand expression

AstExprBinary

ExpressionAST

A binary operator expression: `left op right`. Covers arithmetic, comparison, logical, and concatenation operators.

Example
local sum = a + b
local eq = a == b
local logic = a and b
local concat = a .. b
Fields
op number
The binary operator code (see BinaryOp enum in parser)
left AstExpr
The left operand expression
right AstExpr
The right operand expression

AstExprTypeAssertion

ExpressionTypeAST

A type assertion expression: `expr :: Type`. Used to assert that an expression has a specific type.

Example
local x = y :: number
local z = (a + b) :: any
Fields
expr AstExpr
The expression being asserted
annotation AstType
The type being asserted

AstExprInstantiate

ExpressionAST

An explicit type instantiation expression: `expr<<T, U>>`. Used to explicitly specify type arguments for a generic function or type.

Example
-- Calling a generic function with explicit types
local res = map<<number, string>>(list, tostring)
Fields
expr AstExpr
The expression being instantiated
typeArguments {AstTypeOrPack}
The type arguments provided

AstExprIfElse

ExpressionAST

An if-else expression: `if condition then trueExpr else falseExpr`. The expression form of conditional logic (not the statement form).

Example
local max = if a > b then a else b
local status = if x then "ok" elseif y then "warn" else "error"
Fields
condition AstExpr
The condition expression
hasThen boolean
True if the `then` keyword is present
trueExpr AstExpr
Expression evaluated when condition is truthy
hasElse boolean
True if the `else` keyword is present
falseExpr AstExpr
Expression evaluated when condition is falsy

AstExprInterpString

ExpressionAST

An interpolated string expression: `` `text {expr} more text` ``. Allows embedding expressions directly within string literals.

Example
local s = `Hello {name}, score: {score}`
local t = `Count: {count + 1}`
Fields
strings {string}
The literal string segments between expressions
expressions {AstExpr}
The embedded expressions

AstExprError

ErrorExpressionAST

An error expression node, created when parsing fails. Used to allow partial parsing to continue after errors.

Fields
expressions {AstExpr}
Any expressions that were successfully parsed
messageIndex number
Index into the error messages table

Statements

AstStatBlock

StatementAST

A block of statements, representing a scope boundary. Used for function bodies, control structure bodies, and the top-level chunk.

Example
do
    local x = 1
    print(x)
end
Fields
body {AstStat}
The list of statements in this block
hasEnd boolean
True if the block is terminated by `end` keyword
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatIf

StatementAST

An if statement: `if condition then ... elseif ... else ... end`.

Example
if x > 0 then
    print("positive")
elseif x < 0 then
    print("negative")
else
    print("zero")
end
Fields
condition AstExpr
The condition expression
thenbody AstStatBlock
The block executed when condition is truthy
elsebody AstStat?
Optional else/elseif branch (can be another AstStatIf or AstStatBlock)
thenLocation Location?
Location of the `then` keyword
elseLocation Location?
Location of the `else`/`elseif` keyword (if present)
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatWhile

StatementAST

A while loop: `while condition do ... end`.

Example
while x > 0 do
    x = x - 1
end
Fields
condition AstExpr
The loop condition expression
body AstStatBlock
The loop body block
hasDo boolean
True if the `do` keyword is present
doLocation Location
Location of the `do` keyword
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatRepeat

StatementAST

A repeat-until loop: `repeat ... until condition`. Executes the body at least once, then continues while condition is falsy.

Example
repeat
    x = x + 1
until x > 10
Fields
condition AstExpr
The loop termination condition
body AstStatBlock
The loop body block
hasUntil boolean
True if the `until` keyword is present
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatBreak

StatementAST

A break statement: `break`. Exits the innermost loop immediately.

Example
while true do
    if done then break end
end

AstStatContinue

StatementAST

A continue statement: `continue` (Luau extension). Skips to the next iteration of the innermost loop.

Example
for i = 1, 10 do
    if i % 2 == 0 then continue end
    print(i)
end

AstStatReturn

StatementAST

A return statement: `return expr1, expr2, ...`. Returns values from the current function. Must be the last statement in a block.

Example
return 1, true
return -- void return
Fields
list
The list of expressions to return

AstStatExpr

ExpressionStatementAST

An expression statement: evaluates an expression for its side effects. Typically function calls like `print("hello")`.

Example
print("hello world")
math.randomseed(12345)
Fields
expr
The expression to evaluate

AstStatLocal

StatementAST

A local variable declaration: `local a, b = expr1, expr2`.

Example
local x = 1
local y: number = 2
local a, b = 10, 20
Fields
vars {AstLocal}
The list of local variables being declared
values {AstExpr}
The list of initializer expressions
equalsSignLocation Location?
Location of the `=` sign (if values are provided)
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatFor

StatementAST

A numeric for loop: `for var = from, to, step do ... end`.

Example
for i = 1, 10 do print(i) end
for i = 10, 1, -1 do print(i) end
Fields
var AstLocal
The loop variable
from AstExpr
The starting value expression
to AstExpr
The ending value expression
step AstExpr?
Optional step value expression (defaults to 1)
body AstStatBlock
The loop body block
hasDo boolean
True if the `do` keyword is present
doLocation Location
Location of the `do` keyword
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatForIn

StatementAST

A generic for loop: `for var1, var2 in expr1, expr2 do ... end`. Uses iterators to loop over collections.

Example
for i, v in ipairs(t) do print(i, v) end
for k, v in pairs(t) do print(k, v) end
Fields
vars {AstLocal}
The loop variables
values {AstExpr}
The iterator expressions
body AstStatBlock
The loop body block
hasIn boolean
True if the `in` keyword is present
inLocation Location
Location of the `in` keyword
hasDo boolean
True if the `do` keyword is present
doLocation Location
Location of the `do` keyword
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatAssign

StatementAST

An assignment statement: `var1, var2 = expr1, expr2`.

Example
x = 1
a.b = 2
t[k] = 3
x, y = y, x -- Swap
Fields
vars {AstExpr}
The list of l-value expressions being assigned to
values {AstExpr}
The list of expressions being assigned
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatCompoundAssign

StatementAST

A compound assignment statement: `var += expr`, `var -= expr`, etc.

Example
x += 1
y *= 2
s ..= "suffix"
Fields
op number
The binary operator code (Add, Sub, Mul, etc.)
var AstExpr
The l-value expression being modified
value AstExpr
The expression to apply with the operator
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatFunction

StatementAST

A function declaration statement: `function a.b.c:name(...) ... end`.

Example
function f() end
function t.a() end
function t:m() end
Fields
name AstExpr
The l-value expression where the function is stored
func AstExprFunction
The function expression
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatLocalFunction

StatementAST

A local function declaration: `local function name(...) ... end`. The function is available within its own body (allows recursion).

Example
local function fact(n)
    return n > 0 and n * fact(n-1) or 1
end
Fields
name AstLocal
The local variable holding the function
func AstExprFunction
The function expression
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatTypeAlias

StatementTypeAST

A type alias declaration: `type Name<T> = Type` or `export type Name = Type`.

Example
type Point = {x: number, y: number}
export type Identifier = string | number
Fields
name string
The name of the type alias
nameLocation Location
Location of the type name
generics {AstGenericType}
Generic type parameters
genericPacks {AstGenericTypePack}
Generic type pack parameters
type AstType
The type being aliased
exported boolean
True if this is an `export type` declaration
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatTypeFunction

StatementTypeAST

A type function declaration (Luau extension): `type function Name(...) ... end`. Defines a compile-time type-level function.

Example
type function add(a, b)
    return a + b
end
Fields
name string
The name of the type function
nameLocation Location
Location of the type function name
body AstExprFunction
The function body (as an AstExprFunction)
exported boolean
True if this is an `export type function` declaration
hasErrors boolean
True if parsing encountered errors
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatDeclareGlobal

StatementAST

A global variable declaration (definition file syntax): `declare name: Type`.

Example
declare globalVar: number
declare console: { log: (...any) -> () }
Fields
name string
The name of the global variable
nameLocation Location
Location of the variable name
type AstType
The declared type of the global
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatDeclareFunction

StatementAST

A function declaration (definition file syntax): `declare function name(...)`. Note: Primarily used for type definition files.

Example
declare function print(...: any): ()
declare function map<T, U>(list: {T}, fn: (T) -> U): {U}
Fields
attributes {AstAttr}
Function attributes
name string
The function name
nameLocation Location
Location of the function name
generics {AstGenericType}
Generic type parameters
genericPacks {AstGenericTypePack}
Generic type pack parameters
params AstTypeList
The parameter types
paramNames {AstArgumentName}
The parameter names (for documentation)
vararg boolean
True if function accepts varargs
varargLocation Location
Location of vararg `...` token
retTypes AstTypePack
The return type pack
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatDeclareExternType

StatementTypeAST

An extern type declaration (definition file syntax): `declare extern type Name`. Used to declare external classes/types in definition files.

Example
declare extern type Part
declare extern type Workspace extends Instance
Fields
name string
The name of the extern type
superName string?
Optional parent type name (inheritance)
props {AstDeclaredExternTypeProperty}
Properties and methods of the extern type
indexer AstTableIndexer?
Optional indexer signature
hasSemicolon boolean?
True if followed by an optional semicolon

AstStatError

ErrorStatementAST

An error statement node, created when parsing fails. Used to allow partial parsing to continue after errors.

Fields
expressions {AstExpr}
Any expressions that were successfully parsed
statements {AstStat}
Any statements that were successfully parsed
messageIndex number
Index into the error messages table
hasSemicolon boolean?
True if followed by an optional semicolon

Type Annotations

AstTypeReference

TypeAST

A type reference: `TypeName` or `Module.TypeName<Params>`.

Example
local x: number
local y: string
local z: MyModule.CustomType
local list: Array<number>
Fields
hasParameterList boolean
True if type parameters are provided (even if empty)
prefix string?
Optional module/namespace prefix
prefixLocation Location?
Location of the prefix
name string
The type name
nameLocation Location
Location of the type name
parameters {AstTypeOrPack}
Type arguments passed to generic types

AstTypeTable

TypeAST

A table type: `{ prop: Type, [Key]: Value }`.

Example
type Point = { x: number, y: number }
type Dictionary = { [string]: number }
type Mixed = { name: string, [number]: boolean }
Fields
props {AstTableProp}
The named properties of the table type
indexer AstTableIndexer?
Optional indexer signature `[KeyType]: ValueType`

AstTypeFunction

TypeAST

A function type: `(Arg1, Arg2) -> (Ret1, Ret2)`.

Example
type Callback = (number, string) -> boolean
type Transform<T, U> = (T) -> U
type Predicate = (value: any) -> boolean
Fields
attributes {AstAttr}
Function type attributes
generics {AstGenericType}
Generic type parameters
genericPacks {AstGenericTypePack}
Generic type pack parameters
argTypes AstTypeList
The parameter types
argNames {AstArgumentName?}
Optional parameter names (for documentation)
returnTypes AstTypePack
The return type pack

AstTypeTypeof

TypeAST

A typeof type: `typeof(expr)`. Infers the type from the given expression at compile time.

Example
local x = {a = 1, b = "hello"}
type MyType = typeof(x)

local function getValue()
    return 42
end
type RetType = typeof(getValue())
Fields
expr AstExpr
The expression whose type is inferred

AstTypeUnion

TypeAST

A union type: `Type1 | Type2 | Type3`. Represents a value that can be any one of the constituent types.

Example
type StringOrNumber = string | number
type Result<T> = T | nil
type Status = "pending" | "success" | "error"
Fields
types {AstType}
The types in the union

AstTypeIntersection

TypeAST

An intersection type: `Type1 & Type2 & Type3`. Represents a value that is all of the constituent types simultaneously.

Example
type Named = { name: string }
type Aged = { age: number }
type Person = Named & Aged  -- has both name and age
Fields
types {AstType}
The types in the intersection

AstTypeSingletonBool

SingletonTypeAST

A boolean singleton type: `true` or `false` as a type. Represents a type that can only be one specific boolean value.

Example
type AlwaysTrue = true
type SuccessFlag = true | false
Fields
value boolean
The boolean value (true or false)

AstTypeSingletonString

SingletonTypeAST

A string singleton type: `"literal"` as a type. Represents a type that can only be one specific string value.

Example
type Direction = "north" | "south" | "east" | "west"
type HttpMethod = "GET" | "POST" | "PUT" | "DELETE"
Fields
value string
The string value

AstTypeGroup

TypeAST

A parenthesized type: `(Type)`. Used for grouping in complex type expressions.

Example
type Callback = (string | number) & { name: string }
type UnionGroup = (A | B) & C
Fields
type AstType
The inner type

AstTypeError

ErrorTypeAST

An error type node, created when parsing fails. Used to allow partial parsing to continue after errors.

Fields
types {AstType}
Any types that were successfully parsed
isMissing boolean
True if the type was completely missing
messageIndex number
Index into the error messages table

AstTypeOptional

TypeAST

A special optional type marker used internally. Represents the `?` suffix in optional types like `Type?`.

Example
type optionalString = string?
Fields
location Location
The location of the `?` location

Type Packs

AstTypePackExplicit

PackTypeAST

An explicit type pack: `(Type1, Type2, ...T)`. Represents an explicit list of types, optionally ending with a variadic type.

Example
-- Return type packs
local function multi(): (number, string)
    return 1, "hello"
end
Fields
types {AstType}
The list of types in the pack
tailType AstTypePack?
Optional variadic tail type

AstTypePackVariadic

PackTypeAST

A variadic type pack: `...Type`. Represents zero or more values of the specified type.

Example
local function sum(...: number): number
    -- ... accepts any number of number arguments
end

type VarArgs = (...string) -> ()
Fields
variadicType AstType
The type of each variadic element

AstTypePackGeneric

PackTypeAST

A generic type pack reference: `T...`. References a generic type pack parameter.

Example
type Call<Args..., Ret...> = (Args...) -> Ret...

local function wrap<T...>(fn: (T...) -> T...): (T...) -> T...
    return fn
end
Fields
genericName string
The name of the generic type pack
